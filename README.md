# Bike-Store-Data-Analysis-Project-

# Project Overview

* This project analyzes sales, customer behavior, product performance, and store operations for a Bike Store using SQL.
* The goal is to answer key business questions through data-driven insights — helping management improve sales, marketing, and inventory decisions.

## Ojective
* To analyze and understand:
* Which customers and stores perform best
* What products sell the most
* How orders, profit, and inventory vary
* Which areas need operational improvement

## Dataset Description

* The project uses 9 tables representing real-world retail data:
## Table Name	Description
* products-	Product details such as name, brand, category, and price
* stocks-	Quantity of each product available at each store
* brands-	Brand details
* categories-	Product category information
* customers-	Customer contact and location details
* order_items-	Individual items sold within each order
* stores-	Store information including location and contact
* staffs-	Sales staff details
* orders-	Order-level details like date, customer, and staff involved
  
## Tools & Skills Used
* SQL- PostgreSQL 
* Joins, Group By, CTEs, Window Functions, Aggregations, and Ranking
* Data Cleaning & Business Insights Generation

## Key Insights Summary
* Loyal customers drive repeat business — focus on customer retention.
* Some stores and staff perform significantly better — useful for benchmarking.
* Sales fluctuate by month — leverage seasonal marketing.
* Certain categories and products dominate revenue — optimize inventory accordingly.
